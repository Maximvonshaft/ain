# 系统优化梳理

## 性能优化方向
- **首页数据加载存在放大效应**：当前首页会一次性拉取全部条目，并针对每个条目额外查询时间排序的步骤与附件，随着数据量增长，这一批量加载会导致数据库瞬时压力和 PHP 端内存占用双重上升。建议为列表引入分页/无限滚动，并按需异步加载步骤与附件，或增加缓存层减少重复组装。【F:memo.php†L11568-L11621】
- **分类统计可缓存/预聚合**：每次请求都会对分类与 mindmap 进行完整聚合查询（包含对 items 的全表扫描）。可以考虑将结果缓存在 SQLite 内（例如触发器维护计数字段）或加上轻量级的 in-memory 缓存，降低高频页面刷新时的重复计算成本。【F:memo.php†L299-L324】
- **导入流程对分类的逐条查询**：导入 JSON 时，对每一个新出现的分类名称都会执行一次 `INSERT OR IGNORE` + `SELECT id`，当导入批次较大时会显著放慢事务。可改为在遍历前收集所有待导入分类，批量插入后一次性刷新映射表，或使用 `RETURNING` 直接拿到新 ID。【F:memo.php†L845-L885】
- **查询条件缺少覆盖索引**：首页查询会同时过滤 `done`、`category_id`、模糊搜索并按 `order_index` / `updated_at` 排序，当前数据库仅有单列索引，无法有效利用排序字段。可以补充形如 `(done, order_index, updated_at DESC, id DESC)` 及 `(category_id, done, order_index, updated_at DESC)` 的复合索引，或根据访问模式拆分视图以减少排序成本。【F:core/DB.php†L48-L88】【F:memo.php†L11568-L11576】
- **Mindmap 数据一次性嵌入页面**：思维导图列表会把全部 mindmap JSON 嵌入 `<script type="application/json">` 中输出，在导图较多时会显著增加首屏传输体积，建议改为按需请求或仅输出精简摘要字段，再通过异步接口获取完整内容。【F:memo.php†L11538-L11545】

## 前端体验优化方向
- **大量内联 CSS/JS 降低缓存收益**：多个视图都内联了完整的样式与脚本，导致样式重复、无法复用浏览器缓存，后续维护时也难以拆分模块。可以把公共组件（按钮、布局、弹窗等）抽离为独立样式文件与模块化脚本，结合构建流程做按需加载。【F:memo.php†L1334-L1418】【F:memo.php†L11180-L11380】【F:memo.php†L11668-L11810】
- **复杂渐变与混合模式影响渲染性能**：页面背景与卡片使用多重渐变、`mix-blend-mode`、重度阴影，对低性能设备来说会拖慢首次绘制与滚动。建议在暗色主题下保留必要的视觉元素，其他效果改用静态纹理或纯色，以降低 GPU 合成负担。【F:memo.php†L1378-L1394】【F:memo.php†L11180-L11229】
- **列表无延迟加载机制**：当前 DOM 一次性渲染所有卡片并绑定大量事件处理器，数据一多就会阻塞主线程。可以引入虚拟列表、IntersectionObserver 延迟渲染，或将批量事件处理改为事件委托，减少节点数量与监听器数目。【F:memo.php†L11568-L11644】【F:memo.php†L11668-L11818】
- **思维导图脚本体积大且耦合紧密**：导图导入/导出逻辑直接写在页面脚本中，难以复用，且加载即执行。建议拆分为模块（例如使用 ES modules 或构建工具拆包），并在进入导图视图时按需加载，提高首屏加载速度。【F:memo.php†L11686-L11860】

## 样式与设计优化方向
- **配色与可访问性**：主题色大量使用高饱和金色与低对比文本，部分文本（如淡金色徽章、浅灰副标题）在暗背景下对比不足，应根据 WCAG 调整亮度或提供高对比模式，同时校准字号与字距以提升可读性。【F:memo.php†L1334-L1406】【F:memo.php†L11218-L11245】
- **重复类定义导致样式漂移**：`btn` 等组件在不同视图内重复定义且属性不一致，容易产生风格差异。建议建立统一的设计系统变量与组件库，减少重复定义，并在构建流程中共享变量（如 CSS 自定义属性或预处理器）。【F:memo.php†L1401-L1406】【F:memo.php†L11201-L11216】
- **内联 style 影响维护**：大量元素直接写行内样式（如 `display:flex`、`margin`），使得后续响应式调整困难。可以将这些内联样式抽成语义化类名，集中在样式表里维护，同时方便媒体查询与主题切换。【F:memo.php†L11584-L11605】
- **信息层级与留白**：卡片和侧边栏阴影/描边较多，视觉噪点高。可通过减少装饰性边框、增加留白与分组标题来提高信息层级清晰度，结合字体粗细变化和网格系统，让内容更聚焦。【F:memo.php†L1397-L1418】【F:memo.php†L11220-L11252】

以上建议可作为后续规划的参考，优先从数据加载与资源拆分入手，既能减轻服务器压力，也能改善前端体验。
