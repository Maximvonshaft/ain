# 匹配引擎逻辑说明

本文档总结 `apps/order-matcher` 工具的核心匹配策略，便于后续维护与调参。

## 1. 输入数据

- **订单表**：默认第一张表，列 `A`=订单号、`BK`=城市、`BL`=省份；支持通过配置修改列字母。
- **映射表**：默认第一张表，列 `D`=市、`E`=省；同样支持列字母自定义。
- **配置项**：阈值、权重、别名表、噪音词、候选上限、是否要求市省同时命中、抽检比例等，全部集中在 `config.json` 中。

## 2. 归一化

订单侧与映射侧使用相同的标准化流程：

1. `mb_strtolower` 小写化 + `Normalizer::normalize` 去重音，必要时回退 `iconv`。
2. 去掉配置中的噪音词（如 “city of”、“municipality of” 等）。
3. 只保留字母、数字及配置允许的字符，其余转换为空格。
4. 折叠多余空格。
5. 别名折叠：把归一化后的字符串映射到配置的 `aliases` 词表（支持同义组与显式映射）。
6. Token 化：按空格拆分为独特 token，供 Jaccard 相似度使用。

该流程极大缓解了错拼、变音符号、大小写、语言混写带来的噪声。

## 3. 候选削减

- 预处理映射表时，会按城市、省份的前两个字符建立桶（`cityBuckets` / `provinceBuckets`）。
- 匹配订单时，优先在相同前缀的桶中取候选；若为空，则回退到全表。
- 可通过 `max_candidates` 控制每次最多比较的候选数，保证性能线性可控。

## 4. 打分体系

对城市与省份分别计算：

- **基础相似度** `S_base`：`similar_text` 百分比与 `levenshtein` 归一化得分取较大值。
- **Token/Jaccard** `S_tok`：将归一化后的 token 集合做交并比。
- **组合得分**：`Score = 0.7 * S_base + 0.3 * S_tok`（权重可配）；若命中别名加 `alias_bonus` 奖励，封顶 1.0。

订单行的 `score_max` 取城市、省份两者的最大值。

## 5. 置信度分层

使用配置阈值（默认 High≥0.85，Pass≥0.75，Low≥0.65）：

- **High**：`score_max ≥ high`，直接进入主结果。
- **Mid**：`pass ≤ score_max < high`，进入主结果并打“⚠中置信”。
- **Low**：`low ≤ score_max < pass` 或未满足“市/省同时命中”的要求，进入复核池。
- **Fail**：`score_max < low` 或无候选，记入异常列表。

若 `require_both_match=true`，需城市与省份同时≥`pass` 才算命中；否则只需任一命中。

## 6. 冲突处理与标注

- 当候选 Top1 与 Top2 的 `score_max` 差值 < 0.03，视为“多候选冲突”，强制降为 Low 并记录在 `notes`。
- 若城市得分高（≥High）但省份明显不一致（<0.5），标注“省份待核”；反向同理。
- 主结果附带 `match_source` 字段（`city`、`province`、`city+province`、`none`），明确命中来源。

复核文件会展示 Top3 候选及其得分，方便人工判读。

## 7. 统计与告警

运行结束会生成 `run_log.json`，包括：

- 去重前/后订单数、高/中/低/Fail 数量与比例。
- 平均城市/省份/最高得分。
- 缺失地址数量、Top 未命中原因。
- 城市/省份列的有效值比例。
- 告警规则：Low 比例 >15%、High 比例 <50%、列有效值不足阈值等。

控制台同时输出摘要与告警提示，便于快速获知运行质量。

## 8. 复核与抽检

- `低置信度待人工复核.csv`：收集 Low 档及多候选冲突条目。
- `中置信抽检样本.csv`：按 `sampling_ratio`（默认 3%）从 Mid 档随机抽样。
- 复核输出包含原值、匹配对象、双分值、得分来源、候选 Top3 及运行元数据。

## 9. 后续扩展建议

- 如需支持更复杂的词表治理，可在 `config.json` 中追加别名列映射或合法市-省组合校验规则。
- 大批量数据场景可考虑流式读取/写出或拆分并行执行（当前结构已按候选桶划分，易于扩展）。
- 若要输出 XLSX，可在导出层替换/扩展为 `PhpSpreadsheet` 等库。
