# 订单地名匹配引擎说明

本说明文档梳理 `apps/order-matching` 小应用的匹配逻辑、输入输出契约以及运行流程，以便后续扩展与治理。

## 1. 目标与范围

* **目标**：将订单表中的城市（默认 BK 列）/省份（默认 BL 列）与地名映射表中的市（默认 D 列）/省（默认 E 列）做相似度匹配，命中即输出“有效订单”，并提供命中来源及置信度分层。
* **范围**：支持中英文及阿尔巴尼亚语混写，能够容忍错拼、重音符号、别名等差异；在保证性能的前提下提升召回率与精准度，并对低置信度结果沉入灰度复核池。

## 2. 输入 / 输出

### 输入

1. **订单表 CSV**：首行作为表头，至少包含订单号列（默认 A）、城市列（BK）、省份列（BL）。
2. **映射表 CSV**：首行作为词表表头，至少包含城市列（默认 D）、省份列（E）；可通过 `--map-city-alias-cols` / `--map-province-alias-cols` 传入别名列。
3. **运行参数**：通过命令行传入（见 §7），包括相似度阈值、市省命中策略、输出目录等。

### 输出

* `命中订单_*.csv`：高/中置信度订单清单，包含订单原值、匹配对象、双分值、来源、置信度、命中标签与运行配置。
* `低置信度待人工复核_*.csv`：灰度池，逐行列出候选 Top3，便于人工核查与词表扩展。
* `未命中订单_*.csv`：低于低置信阈值的订单，标记未命中原因，供异常分析。
* `异常订单_*.csv`：缺失地址、列缺失等输入异常。
* `运行摘要_*.json`：核心统计及 Top 未命中原因，并在控制台打印汇总。

所有文件默认落地到 `apps/order-matching/output/`，可通过 `--output-dir` 覆盖。

## 3. 字段映射与校验

* 默认列映射遵循 Excel 列字母（A/BK/BL/D/E），可通过命令行参数重写并持久化。
* 启动时校验关键列是否存在；若订单城市与省份同时为空，则直接标记为异常，不参与匹配。
* 订单按传入行顺序处理；如需去重，可在外部清洗或后续扩展。

## 4. 归一化策略

匹配前对订单侧与映射侧统一做标准化，核心规则：

1. 小写化（支持 `mb_strtolower`）。
2. Unicode 归一化并去除重音、音标（Transliterator + `iconv`，同时手动兜底常见字符）。
3. 去除括号、连字符、标点，仅保留字母/数字/空格。
4. 应用内置别名表（如 `Tiranë → Tirana`、`Prishtinë → Pristina` 等）。
5. 清洗行政噪声词（City/Municipality/Region/Prefecture/County 等多语言关键词）。
6. 合并多余空格，最终得到标准化字符串及 token 集（去重后用于 Jaccard 计算）。

归一化与阈值共同决定匹配“松紧度”，可通过扩展别名表或噪声词列表增强效果。

## 5. 匹配引擎

### 5.1 候选集生成

* 根据标准化后的首字符 / 前两个字符建立城市、省份桶。
* 每条订单先基于城市、省份桶召回候选；若归一化为空则退化为全表遍历。

### 5.2 评分组件

对城市与省份分别计算：

* **基础相似度** `S_base`：使用 `similar_text` 归一化得分（0~1）。
* **Token 相似度** `S_tok`：基于 token 集合的 Jaccard 系数（交/并）。
* **组合分**：`Score = 0.7 * S_base + 0.3 * S_tok`，若命中别名列额外 +0.05（不超过 1.0）。

### 5.3 判定与标签

* 默认规则：城市或省份任一得分 ≥ τ（`--threshold`，默认 0.75）即视为命中；可通过 `--require-both` 改为“双字段同时命中”。
* `source` 标记分值更高的一方（city / province），若强制双命中则在未满足时标记 `both-required`。
* 若城市 ≥ 高置信阈值且省份明显偏低（<0.5），自动添加“省份待核”标签，反之亦然。
* 候选 Top2 分差 < 0.03 视为多义冲突，进入灰度并在统计中累积。

## 6. 置信度分层

根据 `--high-threshold`（默认 0.85）、`--mid-threshold`（默认同阈值 τ）、`--low-threshold`（默认 0.65）划分：

* `High`: `score ≥ high` → 直接通过。
* `Mid`: `mid ≤ score < high` → 通过但可抽检。
* `Low`: `low ≤ score < mid` → 进入“低置信度待复核”灰度池，不计入主导出。
* `Fail`: `score < low` → 进入未命中列表，并记录未命中原因（无候选、阈值过低等）。

## 7. 命令行参数

```bash
php run.php --orders=订单.csv --mapping=映射.csv \
  [--output-dir=apps/order-matching/output] \
  [--threshold=0.75 --high-threshold=0.85 --mid-threshold=0.75 --low-threshold=0.65] \
  [--allow-partial|--require-both] \
  [--export-details] \
  [--order-id-col=A --city-col=BK --province-col=BL] \
  [--map-city-col=D --map-province-col=E] \
  [--map-city-alias-cols=F,G --map-province-alias-cols=H]
```

* `--export-details` 会在命中结果中附带标准化后的城市/省份，便于排查。
* `config_id`（结果文件中展示）由上述参数 Hash 生成，用于追溯运行配置。

## 8. 统计与告警

* 控制台输出：总行数、参与匹配行数、各置信度占比、多义冲突次数、Top 未命中原因。
* `运行摘要_*.json` 持久化同样的统计，后续可接入监控。
* 当强制双命中但命中率下降时，`fail_reasons_top` 会暴露 `both_required`，提醒调参。

## 9. 复核辅助

灰度文件中为每条订单列出候选 Top3（含市/省原值、双分值、来源、别名触发情况），运营可直接补充映射词表或更新别名表。随着词表完善，可逐步收敛灰度池。

## 10. 扩展建议

* **别名维护**：根据 `低置信度` 与 `未命中` 文件里的高频地名，更新 `Normalizer::ALIAS_MAP` 或追加映射表别名列。
* **噪声词调整**：`Normalizer::NOISE_WORDS` 支持多语言，可按需扩展。
* **性能**：若映射表巨大，可进一步引入多线程或数据库索引；当前桶过滤保证在中等数据量下基本线性。
* **可视化**：可在此工具基础上扩展轻量后台，接入运行日志与抽检流程。

---

如需新增小功能，可在 `apps/` 下创建子目录并沿用本模式（独立的 `run.php` + 说明文档）。
